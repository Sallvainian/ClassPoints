## 2026-05-31 — Update (schema-choice open question resolved by user)

Decision: use **Zod**. Rationale (user): the schema becomes the single source of truth — derive the TypeScript type (`z.infer`) and the runtime check from one definition so they can't drift; hand-rolled guards re-state the shape twice and rot. Validation surface is small (`layout_data` + the two realtime domains), so bundle cost is negligible. CAP-1/CAP-2, the Constraints, and the success signal were updated to specify Zod and `z.infer`-derived types; the open question was removed. Noted as a **conscious dependency exception**: Zod crosses the PRD's "no new libraries" line, but that line was scoped to the migration phases and #15 is a separate quality item — captured as a Constraint so it isn't read as precedent for the migration hooks.
